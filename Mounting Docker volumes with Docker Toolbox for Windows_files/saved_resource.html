        Hi Ricardo,

        Since you mounted a subdirectory ("C:\Users") instead of the root of your drive ("C:\"), your command should probably be:
        docker run -d -p 7200:7200 -v /Users:/Users --name graphdb-prue ontotext/graphdb:9.5.1-free

        Be careful with letter casing, since the docker machine is running case-sensitive linux.

                    Dejan Geci Mod Jim C 3 years ago edited

                    Docker Desktop and Toolbox work differently. Since Toolbox uses VirtualBox, that machine is put on the network with the host machine, and will have it's own IP. You can do one of the things below:

                    1. Update your hosts file (C:\Windows\System32\drivers\etc\hosts) by adding this line:
                    192.168.99.100 SOME-ALIAS-INDEPENDENT-OF-IP

                    Replace the IP with the correct docker host IP, and then you will be able to use SOME-ALIAS-INDEPENDENT-OF-IP as the host name.

                    2. Use PowerShell to get the IP dynamically and call the command (also possible with cmd but but I don't know the syntax):
                    $docker_machine_ip = (docker-machine ip default)
                    cmd.exe /c "kafka-topics.bat --create --bootstrap-server $($docker_machine_ip):9092 --replication-factor 1 --partitions 1 --topic test"

                    3. Keep it simple and just use the IP

                          Dejan Geci Mod Jim C 4 years ago

                          If I remember correctly, 'C:\Users' is shared by default, so that is probably why it works. If you carefully follow the steps from the post, you will share the root of your drive, and everything should work fine as long as you try to mount volumes on that same drive. Docker Compose can work with relative paths without conversion, so "- ./src:/src" volume will mount the "src" directory from current directory where "docker-compose.yml" is located into "/src" inside the container.

                          As for customizing the VM, you can remove and recreate the "default" machine with additional parameters:
                          docker-machine rm default

                          docker-machine create \
                          --driver virtualbox \
                          --virtualbox-memory 768 \
                          --virtualbox-cpu-count 1 \
                          --virtualbox-share-folder d:\\:d \
                          --virtualbox-no-vtx-check \
                          default
